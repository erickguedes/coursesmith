# CourseSmith Architecture

Version: 1.0

Status: Draft

---

# Overview

CourseSmith is an AI-native framework for curriculum engineering.

The framework is composed of a small execution core and a collection of discoverable plugins.

The core never contains educational knowledge.

Educational knowledge lives inside Agents, Skills, Templates and Pipelines.

---

# Architectural Principles

The architecture follows six principles.

## 1. Runtime First

The runtime only executes workflows.

It never contains educational logic.

---

## 2. Plugin First

Everything is a plugin.

Agents are plugins.

Skills are plugins.

Templates are plugins.

Pipelines are plugins.

Publishers are plugins.

---

## 3. Artifact Driven

Agents communicate exclusively through artifacts.

No agent communicates directly with another agent.

---

## 4. Declarative

Every workflow is described using YAML.

No pipeline should require source-code modifications.

---

## 5. Git Native

Everything generated by CourseSmith must be stored as plain text.

Preferred formats:

* Markdown
* YAML
* JSON

---

## 6. Runtime Agnostic

The framework must not depend on a specific AI runtime.

Adapters may exist for:

* OpenCode
* Claude Code
* Codex CLI
* Cursor
* VSCode
* CLI

# Core Components

The framework consists of the following components.

## Runtime

Responsible for bootstrapping execution.

Responsibilities:

* load configuration
* discover plugins
* validate project
* start pipeline

---

## Configuration Loader

Reads:

coursesmith.yaml

Responsible for:

* project metadata
* selected pipeline
* runtime settings
* output directory
* enabled plugins

---

## Plugin Registry

Discovers plugins automatically.

Supported locations:

```text
.coursesmith/

plugins/

agents/

skills/

templates/

pipelines/
```

Plugins can also be installed from remote repositories.

---

## Pipeline Engine

Executes workflow definitions.

Example:

```yaml
pipeline:

- parser

- curriculum-architect

- lesson-planner

- teacher

- quiz-generator

- publisher
```

The engine knows nothing about education.

It only executes steps.

---

## Artifact Bus

Central communication mechanism.

Every agent receives artifacts.

Every agent produces artifacts.

Example:

course.yaml

↓

module.yaml

↓

lesson-plan.json

↓

lesson.md

↓

quiz.json

↓

flashcards.json

↓

publisher

The Artifact Bus is the only communication channel.

---

# Plugin Model

Every plugin follows the same structure.

```text
plugin/

plugin.yaml

README.md

src/

tests/
```

plugin.yaml describes:

* name
* version
* author
* capabilities
* supported artifacts
* dependencies

# Installation Model

CourseSmith supports two installation modes.

## Global Installation

Recommended for CLI usage.

```bash
npm install -g @coursesmith/cli
```

or

```bash
npx @coursesmith/cli init
```

This installs only the runtime.

No educational plugins are included.

---

## Project Installation

Recommended for reproducible projects.

```bash
npm install --save-dev @coursesmith/core
```

Project structure:

```text
my-course/

coursesmith.yaml

package.json

.coursesmith/
```

The project becomes self-contained.

Every contributor uses the same versions.

---

# Plugin Installation

Plugins are installed per project.

Example:

```bash
npx coursesmith add @coursesmith/teacher
```

or

```bash
npx coursesmith add github:user/plugin-name
```

The command:

downloads the plugin

validates compatibility

registers it

updates coursesmith.yaml

---

# Project Initialization

Creating a new project:

```bash
npx @coursesmith/cli init
```

Generates:

```text
.coursesmith/

agents/

skills/

templates/

pipelines/

coursesmith.yaml

content/
```

# Design Decision

Educational logic MUST NEVER exist inside the Runtime.

The Runtime only:

loads plugins

loads configuration

executes pipelines

routes artifacts

publishes outputs

Everything else belongs to plugins.

---

# Future Runtime Adapters

Future adapters may include:

Claude Code

Cursor

OpenCode

Codex CLI

REST API

Docker

GitHub Actions

The plugin ecosystem remains unchanged regardless of the runtime.

---

# Architectural Summary

CourseSmith is composed of:

A lightweight execution runtime

A discoverable plugin ecosystem

A declarative pipeline engine

An artifact communication bus

A Git-native output model

A runtime-independent architecture

These principles ensure that CourseSmith remains extensible, portable and maintainable as the ecosystem evolves.
